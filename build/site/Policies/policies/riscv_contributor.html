Rationale

The RISC-V membership agreement covers IP topics. If contributors are not members, then they are not bound by a technical contribution agreement.

Policy

  1. Contributors to RISC-V specification text must have signed a member agreement.

  2. RISC-V staff will set the github maintainers to the chairs and vice chairs only. Additional information can be found in the GitHub Repo Structure & Administration policy.

  3. Task Groups and Committees developing specifications should not allow any pull requests from non-members (RISC-V staff will help likely using the certificate of origin used by the linux kernel).

  4. An audit must be done before a specification goes to public review as to whether any pull requests got into the specification by non-members. RISC-V staff will work to identify or develop utilities to assist in this process

  5. The CTO will sign-off that the review has been done as part of DoD.

Exception handling

Not applicable

Transition to start using policy

Immediately
